# 模拟股票交易对敲撮合系统 — 项目使用说明

本文档说明如何在本机**安装环境、启动服务、访问地址、基本使用流程**及注意事项。更详细的模块说明与接口定义见 `docs/` 下各模块文档。

---

## 一、环境准备

| 项目 | 要求 | 说明 |
|------|------|------|
| JDK | **17** | 全项目统一，exchange-core 依赖要求 |
| Maven | 3.6+ | 编译与打包 |
| MySQL | 5.7 或 8.x | 仅 trading-admin 使用；需先创建数据库 `trading_admin` |
| exchange-core | 已安装到本地 Maven | 需在引擎项目目录执行 `mvn install`，否则 trading-core 无法编译 |

**检查命令：**

```bash
java -version   # 应显示 17.x
mvn -version
```

**MySQL：** 创建库 `trading_admin`，并在 `trading-admin/src/main/resources/application.yaml` 中配置 `spring.datasource`（密码可通过环境变量 `DB_PASSWORD` 配置）。

## 二、启动方式

### 2.1 一键启动（推荐）

在项目根目录执行：

**Mac / Linux：**

```bash
./start-all.sh
```

- 会先执行 `mvn clean package -DskipTests`，再依次启动 **trading-admin(8080)** → **trading-core(8081)** → **trading-gateway(9000)**，并等待各端口就绪。
- 进程 PID 写入 `.trading-system.pids`，日志输出到根目录：`logs-admin.log`、`logs-core.log`、`logs-gateway.log`。
- **停止全部**：`./stop-all.sh`。

**Windows：**

```cmd
start-all.bat
```

- 会在三个新窗口中分别启动 Admin、Core、Gateway；关闭对应窗口即可停止该服务，或运行 `stop-all.bat`。

**注意：** 一键启动前请确保 MySQL 已启动并已创建 `trading_admin`，且 exchange-core 已 `mvn install`。

### 2.2 分步启动

1. 启动 MySQL，并配置 Admin 的 `application.yaml`。
2. 启动 **trading-admin**：`cd trading-admin && mvn spring-boot:run`（端口 8080）。
3. 启动 **trading-core**：`cd trading-core && mvn spring-boot:run`（端口 8081）。  
   - 若需兼容 exchange-core，可加 JVM 参数：  
     `-Dspring-boot.run.jvmArguments="--add-opens=java.base/sun.nio.ch=ALL-UNNAMED --add-opens=java.base/java.nio=ALL-UNNAMED"`
4. 启动 **trading-gateway**：`cd trading-gateway && mvn spring-boot:run`（TCP 9000，HTTP 8082 可选）。

## 三、访问地址与端口（默认）

| 服务 | 地址 | 说明 |
|------|------|------|
| trading-admin | http://localhost:8080 | 用户管理、订单历史查询、内部订单事件接口 |
| trading-core | http://localhost:8081 | 下单、撤单、查询挂单 |
| trading-gateway | TCP localhost:9000 | 客户端按行 JSON 下单/撤单 |
| 健康检查 | http://localhost:8080/actuator/health 等 | Admin/Core 8081、Gateway 8082 均有 /actuator/health |
| **前端页面** | 见下方说明 | 可视化操作界面 |

### 3.1 前端页面访问

项目提供了完整的前端页面，位于 `trading-frontend/` 目录。

**启动方式**（任选其一）：

1. **Python HTTP 服务器**（推荐）：
   ```bash
   cd trading-frontend
   python3 -m http.server 8000
   ```
   然后访问：`http://localhost:8000`

2. **Node.js http-server**：
   ```bash
   cd trading-frontend
   http-server -p 8000
   ```

3. **直接打开 HTML**（可能遇到 CORS 问题）：
   直接用浏览器打开 `trading-frontend/index.html`

**前端功能**：
- **系统概览**：服务状态、用户统计、快速操作
- **用户管理**：创建用户（仅填用户名，系统生成 uid/股东号）、修改用户名、查询余额、禁用/启用
- **交易下单**：下单（可选订单号，系统可自动生成）、撤单（仅需原订单编号）
- **订单历史**：查询历史订单（分页、筛选）、**删除单条**、**删除全部**
- **当前挂单**：按股东号查询未撤单订单（成交后自动从挂单移除）

详细使用说明见：`trading-frontend/README.md`

## 四、基本使用流程（新手向）

1. **创建用户（股东）**  
   在 Admin 创建用户，**仅需传用户名**，系统自动生成 uid 与股东号（格式 SH+8 位数字）；Core 会校验股东号存在且启用：
   ```bash
   curl -X POST http://localhost:8080/api/users \
     -H "Content-Type: application/json" \
     -d '{"username":"测试用户"}'
   ```

2. **下单**  
   - 方式一（直接调 Core）：  
     `POST http://localhost:8081/api/order`，Body 含 market、securityId、side、qty、price、shareholderId；clOrderId 可不传，系统自动生成并返回。  
   - 方式二（经 Gateway）：  
     TCP 连 9000，发送一行订单 JSON，收到一行回报 JSON。

3. **撤单**  
   - 方式一：`POST http://localhost:8081/api/cancel`，Body 仅需 **origClOrderId**（原订单编号）；其余可从挂单记录自动带出，clOrderId 可不传。  
   - 方式二：TCP 连 9000，发送一行撤单 JSON（含 origClOrderId）。

4. **查订单历史**  
   Admin：`GET http://localhost:8080/api/orders/history?shareholderId=SH00010001&page=0&size=20`（可选 status）。  
   **删除**：`DELETE /api/orders/history/{id}` 删单条；`DELETE /api/orders/history` 删全部。

5. **查当前挂单**  
   Core：`GET http://localhost:8081/api/orders?shareholderId=SH00010001`。成交后订单会从挂单列表自动移除。

**市场与方向约定：** market 仅支持 `XSHG`（上交所）、`XSHE`（深交所）、`BJSE`（北交所）；side 为 `B`（买）或 `S`（卖）。详见 `docs/项目说明书.md` 或各模块文档。

## 五、注意事项

- **无登录鉴权**：当前 REST 接口无鉴权，仅适合内网或本地测试；生产环境需自行增加鉴权。
- **内部接口**：Admin 的 `POST /internal/order-events` 供 Core 推送订单事件用；若配置了 `trading.admin.internal-token`，请求头须带 `X-Internal-Token` 且与配置一致。
- **多机部署**：若 Admin/Core 不在本机，需修改 Core 的 `trading.admin.base-url`、Gateway 的 `trading.gateway.core-base-url`。
