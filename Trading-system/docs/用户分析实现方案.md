## 用户分析模块设计与使用说明

### 一、功能概览

用户分析模块用于对**单个用户在指定时间区间内的交易行为**进行统计与可视化展示，帮助使用者快速了解该用户的交易活跃度、成交质量和资金体量。

**支持的能力包括：**

- **整体统计概览**
  - 订单总数（`totalOrders`）
  - 成交订单数（`filledOrders`）
  - 撤单订单数（`cancelledOrders`）
  - 成交总金额（`totalTurnover`）
  - 成交均价（`avgPrice`）

- **分日趋势分析（`dailyStats`）**
  - 每日订单总数
  - 每日成交订单数
  - 每日撤单订单数
  - 每日成交总金额

- **可视化展示**
  - 成交 vs 撤单占比条形图（当前区间内聚合）
  - 每日订单数量柱状图（总数 / 成交 / 撤单）
  - 每日成交金额折线图

---

### 二、数据来源与统计逻辑（后端）

#### 1. 数据来源

- **表**：`t_order_history`  
- **实体**：`OrderHistory`  

**过滤条件：**

- 按用户 `uid` 找到对应的 `shareholderId`
- 查询该股东号在指定时间区间 `[start, end]` 内的历史订单：

```java
orderHistoryRepository.findByShareholderIdAndCreatedAtBetween(shareholderId, start, end);
```

#### 2. 聚合结果对象 `UserAnalysisDTO`

**字段说明：**

- `totalOrders`：查询到的订单总条数。
- `filledOrders`：`status == "FILLED"` 的订单数。
- `cancelledOrders`：`status == "CANCELLED"` 的订单数。
- `totalTurnover`：对所有有成交数量的订单，按 `price * filledQty` 累加。
- `avgPrice`：若总成交数量 > 0，则 `totalTurnover / 总成交数量`，保留 4 位小数；否则为 0。
- `dailyStats`：`List<DailyStat>`，按订单 `createdAt` 的日期 `LocalDate` 聚合：
  - `date`：日期字符串，如 `2026-03-07`
  - `totalOrders`：当日订单数
  - `filledOrders`：当日成交订单数
  - `cancelledOrders`：当日撤单订单数
  - `totalTurnover`：当日成交金额之和

### 三、接口设计

#### 1. HTTP 接口

- **URL**：`GET /analysis/user/{userId}`
- **查询参数：**
  - `start`：开始时间，`yyyy-MM-dd'T'HH:mm[:ss]` 格式（例如 `2026-03-01T00:00:00`）
  - `end`：结束时间，同上
- **返回体**：`UserAnalysisDTO` 的 JSON 序列化结果

**示例请求：**

```http
GET /analysis/user/1?start=2026-03-01T00:00:00&end=2026-03-07T23:59:59
```

**返回示例：**

```json
{
  "totalOrders": 10,
  "filledOrders": 6,
  "cancelledOrders": 2,
  "totalTurnover": 123456.78,
  "avgPrice": 10.1234,
  "dailyStats": [
    {
      "date": "2026-03-05",
      "totalOrders": 3,
      "filledOrders": 2,
      "cancelledOrders": 1,
      "totalTurnover": 34567.89
    },
    {
      "date": "2026-03-06",
      "totalOrders": 7,
      "filledOrders": 4,
      "cancelledOrders": 1,
      "totalTurnover": 88888.89
    }
  ]
}
```

### 四、前端交互与展示逻辑

#### 1. 页面入口

- **页面**：`user-analysis.html`  
- **脚本**：`js/user-analysis.js`  
- **导航位置**：顶部导航栏「用户分析」

#### 2. 查询流程

1. 用户在“查询条件”区域填写：
   - **用户ID (`uid`)**：系统内部用户 ID。
   - **开始时间 / 结束时间**：通过 `datetime-local` 组件选择。
2. 点击“开始分析”后，前端会：
   - 将 `datetime-local` 值补上秒位（如 `2026-03-02T12:34` → `2026-03-02T12:34:00`）。
   - 调用 `AdminAPI.getUserAnalysis(uid, start, end)` 请求后端接口。
3. 收到结果后：
   - 若成功：更新统计卡片、占比条形图和两张时间序列图。
   - 若失败：清空统计数据，显示错误提示信息。

#### 3. 图表含义

- **成交 vs 撤单占比条形图**
  - 维度：当前时间区间内所有订单。
  - 用成交订单数和撤单订单数占和的百分比展示“成交质量”和“撤单比例”。

- **每日订单数量柱状图（`chart-orders-per-day`）**
  - X 轴：日期  
  - Y 轴：订单数量  
  - 三个柱：
    - 订单总数
    - 成交订单数
    - 撤单订单数
  - 用于观察：某个用户在一段时间内的下单活跃度和成交 / 撤单结构。

- **每日成交金额折线图（`chart-turnover-per-day`）**
  - X 轴：日期  
  - Y 轴：成交金额  
  - 一条折线：每日成交金额  
  - 用于观察：该用户交易资金体量随时间的变化趋势（是否集中在某几天、是否有峰值等）。

### 五、使用建议与典型场景

- **风控 / 运营侧**
  - 查看某用户在一段时间内是否出现异常交易量峰值。
  - 对比成交订单与撤单订单比例，识别下单后频繁撤单的行为。

- **教学 / 演示场景**
  - 通过选择不同时间区间，演示“高频交易”和“低频交易”在图表上的差异。
  - 结合单个用户的订单历史，讲解成交均价、成交金额等概念。

- **调试 / 测试**
  - 在下单 / 撤单后，选择最近一小时或一天做分析，验证统计结果是否与预期一致。
  - 若图表为空或数据明显不对，说明订单历史写入或统计逻辑可能存在问题，可以据此排查。
